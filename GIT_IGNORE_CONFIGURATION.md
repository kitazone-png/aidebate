# Git Ignore Configuration

## Files Created

### 1. `.gitignore`

Comprehensive ignore file for Java Spring Boot project that excludes:

#### Build Artifacts
- `target/` - Maven build output directories
- `*.class` - Compiled Java classes
- `*.jar`, `*.war`, `*.ear` - Package files (except wrapper JARs)
- Maven metadata files

#### IDE & Editor Files
- `.idea/`, `*.iml`, `*.ipr` - IntelliJ IDEA
- `.vscode/` - VS Code settings
- `.settings/`, `.project`, `.classpath` - Eclipse
- `.qoder/` - Qoder IDE workspace files

#### Operating System Files
- `.DS_Store` - macOS metadata
- `Thumbs.db`, `Desktop.ini` - Windows metadata
- `*~`, `.directory` - Linux temp files

#### Application Logs & Data
- `logs/` - Application log directory
- `*.log` - All log files
- `*.db`, `*.sqlite`, `*.h2.db` - Database files

#### Sensitive Configuration
- `.env`, `.env.local` - Environment variables
- `application-local.yml` - Local configuration
- `application-dev.yml` - Development configuration
- `application-prod.yml` - Production configuration
- `*.pem`, `*.key`, `*.jks` - Private keys and certificates

#### Temporary & Cache Files
- `*.tmp`, `*.temp`, `*.cache`
- `*.bak` - Backup files
- `backup/` - Backup directory

#### Project-Specific
- `audio-cache/` - Cached audio files (TTS feature)
- `uploads/`, `user-data/` - User uploads
- `sessions/` - Session data

#### Files Explicitly Kept
- `README.md`, `SETUP.md`, `BUILD.md` - Documentation
- `src/main/resources/static/**` - Static web resources
- `src/main/resources/db/**` - Database migrations
- Configuration templates

### 2. `.gitattributes`

Configures Git's handling of line endings and file types:

#### Text Files (LF line endings)
- Source code: `*.java`, `*.xml`, `*.yml`, `*.sql`
- Web files: `*.html`, `*.css`, `*.js`, `*.json`, `*.md`
- Shell scripts: `*.sh`

#### Text Files (CRLF line endings)
- Windows scripts: `*.bat`, `*.cmd`, `*.ps1`

#### Binary Files
- Archives: `*.jar`, `*.zip`, `*.gz`, `*.tar`
- Images: `*.png`, `*.jpg`, `*.gif`, `*.ico`
- Audio/Video: `*.mp3`, `*.mp4`, `*.wav`
- Documents: `*.pdf`, `*.doc`, `*.docx`
- Fonts: `*.ttf`, `*.woff`, `*.woff2`

## Files Removed from Git Tracking

The following files/directories were removed from Git tracking but remain in the working directory:

### Build Artifacts (All Modules)
```
aidebate-adapter/target/
aidebate-app/target/
aidebate-domain/target/
aidebate-infrastructure/target/
aidebate-start/target/
```

**Reason**: Build artifacts should be regenerated by each developer. They contain compiled `.class` files and Maven metadata that vary by environment.

**Impact**: ~200+ files removed from version control

### IDE Configuration
```
.vscode/
.qoder/
```

**Reason**: IDE-specific settings should not be shared as they may contain local file paths and personal preferences.

**Impact**: Developer-specific settings isolated

### Logs
```
logs/
aidebate-start/logs/
*.log
```

**Reason**: Log files are runtime-generated and can contain sensitive data or become very large.

**Impact**: Repository stays clean, log files generated fresh in each environment

### Backup Files
```
*.bak
*-original.*
```

**Reason**: Backup files are temporary and should not be version controlled.

**Impact**: Cleaner repository structure

## Benefits

### 1. **Smaller Repository Size**
- Excluding build artifacts significantly reduces repository size
- Faster clone and pull operations
- Less storage used on GitHub

### 2. **Cleaner Git History**
- Changes to source code are easier to track
- No noise from auto-generated files
- Meaningful diffs for code reviews

### 3. **No Merge Conflicts on Generated Files**
- Build artifacts won't cause merge conflicts
- IDE settings won't interfere between developers
- Each developer can have their own local configuration

### 4. **Security**
- Private keys and certificates excluded by default
- Environment-specific configurations (with sensitive data) not committed
- API keys and secrets protected

### 5. **Cross-Platform Compatibility**
- `.gitattributes` ensures consistent line endings
- Works correctly on Windows, macOS, and Linux
- No CRLF/LF issues

## Usage

### For New Developers

After cloning the repository:

1. **Build the project** to generate `target/` directories:
   ```bash
   mvn clean compile
   ```

2. **Create local configuration** (not tracked by Git):
   ```bash
   cp src/main/resources/application.yml src/main/resources/application-local.yml
   # Edit application-local.yml with local settings
   ```

3. **Set environment variables** for sensitive data:
   ```bash
   export DASHSCOPE_API_KEY=your-api-key-here
   ```

### Adding New Files

When you create new files:

- **Source files**: Will be tracked automatically (`.java`, `.xml`, `.html`, etc.)
- **Build artifacts**: Automatically ignored
- **Logs**: Automatically ignored
- **IDE files**: Automatically ignored

### Checking Ignored Files

To see what files are ignored:
```bash
git status --ignored
```

To check if a specific file will be ignored:
```bash
git check-ignore -v path/to/file
```

## Maintenance

### Adding New Ignore Patterns

Edit `.gitignore` and add patterns as needed:

```gitignore
# Example: Ignore a new directory
new-directory/

# Example: Ignore a file type
*.extension
```

### Force-Adding an Ignored File

If you need to track a file that's normally ignored:

```bash
git add -f path/to/file
```

**Warning**: Only do this for files like `maven-wrapper.jar` that should be committed.

## Best Practices

### DO Commit:
✅ Source code (`.java`, `.xml`, `.html`, `.js`, `.css`)
✅ Configuration templates (`application-template.yml`)
✅ Database migrations (`schema.sql`, migration files)
✅ Static resources (images, fonts in `src/main/resources/static/`)
✅ Documentation (`.md` files)
✅ Build configuration (`pom.xml`, `package.json`)
✅ Test code and resources

### DON'T Commit:
❌ Build artifacts (`target/`, `*.class`)
❌ IDE settings (`.idea/`, `.vscode/`, `.qoder/`)
❌ Logs (`*.log`, `logs/`)
❌ Environment-specific config (`application-local.yml`)
❌ Sensitive data (API keys, passwords, certificates)
❌ Temporary files (`*.tmp`, `*.bak`)
❌ OS-specific files (`.DS_Store`, `Thumbs.db`)
❌ Dependencies downloaded by Maven (in `target/` or `.m2/`)

## Troubleshooting

### Problem: File is ignored but I want to track it

**Solution**: 
1. Check if it matches an ignore pattern: `git check-ignore -v path/to/file`
2. Either remove the pattern from `.gitignore` or force-add: `git add -f path/to/file`

### Problem: Previously committed file now ignored

**Solution**:
```bash
# Remove from Git tracking but keep in working directory
git rm --cached path/to/file

# Commit the removal
git commit -m "Remove file from tracking (now in .gitignore)"
```

### Problem: Want to see what files are being ignored

**Solution**:
```bash
git status --ignored
```

### Problem: Accidentally committed build artifacts

**Solution**:
```bash
# Remove from Git tracking
git rm -r --cached target/

# Commit the removal
git commit -m "Remove build artifacts from version control"
```
